# Contributing to Awesome AI Vibe Coding

First off, thank you for considering contributing to Awesome AI Vibe Coding! It's people like you that make this resource valuable for the entire developer community.

## üåü How Can I Contribute?

### Reporting Issues

- **Check existing issues** before creating a new one
- Use a **clear and descriptive title**
- Provide **detailed information** about the issue
- Include **steps to reproduce** if applicable

### Suggesting Resources

We love new suggestions! Here's how to suggest a resource:

1. **Check if it already exists** in the README
2. **Verify the resource is active** and maintained
3. **Test the resource** yourself before suggesting
4. **Provide complete information**:
   - Name and link
   - Clear description
   - Why it's valuable
   - Category it belongs to

### Pull Request Process

1. **Fork the repository**
2. **Create a new branch** (`git checkout -b feature/add-awesome-tool`)
3. **Make your changes**
4. **Test all links** to ensure they work
5. **Follow the formatting guidelines** (see below)
6. **Commit with clear messages** (`git commit -m 'Add: Awesome AI Tool'`)
7. **Push to your fork** (`git push origin feature/add-awesome-tool`)
8. **Open a Pull Request**

## üìã Guidelines

### Quality Standards

Resources should be:

- **Active**: Recently updated or maintained
- **Relevant**: Related to AI-assisted coding
- **Accessible**: Publicly available (free or paid)
- **Valuable**: Provides clear benefit to developers
- **Documented**: Has clear documentation or instructions

### Formatting Guidelines

#### Adding a Tool

```markdown
- [Tool Name](https://tool-url.com/) - Brief, clear description of what it does
```

#### Adding a Course/Tutorial

```markdown
- **[Course Name](https://course-url.com/)** - Description, provider, duration, cost
```

#### Adding a Research Paper

```markdown
- [Paper Title](https://paper-url.com) - Brief summary of key findings
```

### Description Guidelines

- **Be concise**: 1-2 sentences maximum
- **Be specific**: What makes it unique?
- **Be objective**: Avoid marketing language
- **Be accurate**: Test before submitting

### Categories

Place resources in the appropriate category:

- **Browser-Based Tools**: Web applications
- **IDEs and Code Editors**: Desktop development environments
- **Mobile Apps**: Mobile applications
- **Plugins and Extensions**: IDE extensions
- **Local Apps**: Desktop applications
- **Command Line Tools**: Terminal-based tools
- **Task Management**: Project management tools
- **Documentation**: Documentation generators
- **Learning Resources**: Courses, tutorials, guides
- **Research Papers**: Academic publications
- **Communities**: Forums, discussion boards
- **News**: Articles, blogs, videos

## üö´ What NOT to Submit

- **Broken links** or inactive projects
- **Duplicate entries** already in the list
- **Self-promotion** without clear value
- **Paid tools** without free tier or trial
- **Unrelated resources** not about AI coding
- **Low-quality** or unmaintained projects

## üí° Suggesting New Categories

If you think a new category would be valuable:

1. Open an issue with the **[Category Suggestion]** tag
2. Explain why the category is needed
3. Provide 3-5 example resources for the category
4. Wait for community feedback

## üîç Review Process

1. **Automated checks**: Links and formatting
2. **Manual review**: Quality and relevance
3. **Community feedback**: 48-hour review period
4. **Merge**: If approved by maintainers

## üìù Commit Message Guidelines

Use clear, descriptive commit messages:

- `Add: [Resource Name]` - Adding new resource
- `Update: [Resource Name]` - Updating existing resource
- `Fix: [Issue]` - Fixing broken links or errors
- `Remove: [Resource Name]` - Removing outdated resource
- `Docs: [Change]` - Documentation updates
